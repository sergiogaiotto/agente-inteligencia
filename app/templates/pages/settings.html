🌐
Idioma de Resposta — Default Global
Aplicado a todos os agentes que não fizeram override no campo response_language. Engine força o LLM a responder neste idioma mesmo quando RAG/MCP retornam conteúdo em outras línguas.

ⓘ Override por agente em Agentes → Editar → step Básico → "Idioma da resposta".

🕒
Timezone da Plataforma
Fuso usado para exibir datas e horas em toda a plataforma (e no horário local do servidor). As datas são armazenadas em UTC — só a exibição muda.

ⓘ Padrão: GMT-3 Brasília. Aplica-se após salvar — recarregue a página para ver as datas no novo fuso.

🛡️
Conhecimento do Modelo — Grounded by Default
Quando ligado, todos os agentes respondem apenas com base em evidências (anexos, base de conhecimento/RAG, resultados de ferramentas). Respostas sem nenhuma evidência são recusadas — o conhecimento geral/paramétrico do modelo NUNCA é usado para inventar.

ⓘ Fonte das evidências: Base de Conhecimento (RAG/Tabelas), anexos ou ferramentas. Esta regra só decide se o agente pode responder sem fonte — não fornece a fonte.

ⓘ Exceção por agente em Agentes → Editar → "Permitir conhecimento geral" (ex.: brainstorming).

🧩
MCP por ferramenta (per-tool) — avançado
Quando ligado, cada ferramenta de um servidor MCP é exposta ao modelo como sua própria função, com o schema real (todos os parâmetros). Sem isso, o servidor inteiro vira uma única função genérica {operation, query} — o que limita ferramentas de argumentos estruturados (ex.: create_issue, tavily_search).

ⓘ Mudança vale em runtime (sem restart). Conectores em Conexões.

💳
Quota de custo por API Key — governança de gasto
Quando ligado, cada invoke feito por uma API Key é debitado do orçamento da chave (custo real de LLM). Ao atingir o teto da janela (dia/mês/acumulado), novos invokes recebem HTTP 402. Sessões de UI (cookie) não são afetadas.

ⓘ Mudança vale em runtime (sem restart). Defina o orçamento por chave em . NB: modelos gpt-oss custam US$ 0 — só Azure/OpenAI acumulam custo.

💬
Tier 2 — Perguntar à Tabela (text-to-SQL governado) — experimental
Quando ligado, habilita a bancada onde a IA compila uma pergunta em português numa consulta estruturada (sem gerar SQL cru), o humano cura e só então ela vira consultável. A governança (mascaramento de PII por coluna, allow-list) vem do Catálogo de Dados da tabela. Diferente do Tier 1 parametrizado, onde o autor já escreve os filtros à mão.

ⓘ Mudança vale em runtime (sem restart). Tabelas e catálogo em Conhecimento.
